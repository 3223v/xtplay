# XTPlay PostgreSQL Storage Design

## Goal

XTPlay previously persisted runtime data in JSON files under `xtplay_b/data`. That made roles, presets, entries, lorebooks, and stories shared by every login user. The PostgreSQL design moves application data into the local `xtplay` database and adds an explicit `user_id` owner to every user-scoped resource.

Documents are frontend content only. They are not stored in PostgreSQL and have no user owner.

## Connection

The backend reads the connection string from `XTPLAY_DATABASE_URL`, then `DATABASE_URL`, then `POSTGRES_DSN`. If none is configured, it uses:

```text
postgresql://postgres:root@localhost:5432/xtplay
```

## Isolation Model

The current prototype login stores the public user in frontend localStorage. Business API calls send `X-User-Id`, and the backend filters every scoped query with `where user_id = :current_user_id`.

This is data isolation for the current lightweight login system, not cryptographic authentication. If the project later adds sessions or JWT, the route dependency can resolve `current_user_id` from the verified token while the tables stay unchanged.

## Table Strategy

The schema uses relational columns for ownership, identity, timestamps, and joins. The existing domain payloads remain in JSONB because the frontend and import formats are already flexible:

- role cards may contain many optional text fields.
- presets are parameter bags and prompt fragments.
- lorebooks contain nested entry maps keyed by uid.
- stories embed selected role/preset/lorebook snapshots.

Stories and story rounds are split into separate tables. A story can grow through many rounds; keeping rounds separate avoids rewriting a large story document for every generated turn and gives a clear place to order or delete individual rounds.

Indexes follow the main access patterns: every user-scoped table has an index on `user_id`, and `story_rounds` also indexes `story_id` plus `(story_id, position, id)` for ordered playback.

## Schema

```sql
create table if not exists users (
  id bigint generated by default as identity primary key,
  username text not null,
  password text not null,
  email text not null default '',
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create unique index if not exists users_username_lower_uidx on users (lower(username));

create table if not exists roles (
  id bigint generated by default as identity primary key,
  user_id bigint not null references users(id) on delete cascade,
  data jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table if not exists presets (
  id bigint generated by default as identity primary key,
  user_id bigint not null references users(id) on delete cascade,
  data jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table if not exists entries (
  id bigint generated by default as identity primary key,
  user_id bigint not null references users(id) on delete cascade,
  data jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table if not exists lorebooks (
  id bigint generated by default as identity primary key,
  user_id bigint not null references users(id) on delete cascade,
  data jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table if not exists stories (
  id bigint generated by default as identity primary key,
  user_id bigint not null references users(id) on delete cascade,
  data jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table if not exists story_rounds (
  id bigint generated by default as identity primary key,
  user_id bigint not null references users(id) on delete cascade,
  story_id bigint not null references stories(id) on delete cascade,
  position integer not null default 0,
  data jsonb not null default '{}'::jsonb,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index if not exists roles_user_id_idx on roles (user_id);
create index if not exists presets_user_id_idx on presets (user_id);
create index if not exists entries_user_id_idx on entries (user_id);
create index if not exists lorebooks_user_id_idx on lorebooks (user_id);
create index if not exists stories_user_id_idx on stories (user_id);
create index if not exists story_rounds_user_id_idx on story_rounds (user_id);
create index if not exists story_rounds_story_id_idx on story_rounds (story_id);
create index if not exists story_rounds_story_position_idx on story_rounds (story_id, position, id);
```

## Legacy Data Bootstrap

At startup the API initializes the schema and seeds existing JSON files into the first user account if the target PostgreSQL tables are empty. The demo `data/users.json` creates the initial `admin` user. Existing role, preset, entry, lorebook, and story JSON files are imported under that first user.

After the tables contain data, startup does not re-import those JSON files.
