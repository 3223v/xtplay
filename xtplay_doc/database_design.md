# XTPlay 数据库设计文档

> 版本 1.1 | 最后更新: 2026-05-12

## 概述

XTPlay 是一个双人小说 AI 生成平台，使用 PostgreSQL 作为主数据库，支持 JSON 格式的灵活数据存储，同时保留 JSON 文件作为遗留数据迁移源。

## 数据库连接

默认连接字符串（可通过环境变量覆盖）：
```env
XTPLAY_DATABASE_URL=postgresql://postgres:root@localhost:5432/xtplay
DATABASE_URL
POSTGRES_DSN
```

## 表结构

### 1. `users` — 用户表

存储系统用户信息，包含角色权限控制。

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | `bigint` | `GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY` | 自增主键 |
| `username` | `text` | `NOT NULL` | 用户名（唯一不区分大小写） |
| `password` | `text` | `NOT NULL` | 密码（明文存储，当前版本） |
| `email` | `text` | `NOT NULL DEFAULT ''` | 邮箱 |
| `role` | `text` | `NOT NULL DEFAULT 'normal'` | 角色：`normal` / `admin` / `super_admin` |
| `created_at` | `timestamptz` | `NOT NULL DEFAULT now()` | 创建时间 |
| `updated_at` | `timestamptz` | `NOT NULL DEFAULT now()` | 更新时间 |

**索引：**
- `users_username_lower_uidx`：`lower(username)` 唯一索引

**角色权限说明：**

| 角色 | 说明 | 权限 |
|------|------|------|
| `normal` | 普通用户 | 管理自己的角色、预设、条目、世界书、故事 |
| `admin` | 管理员 | 普通用户权限 + 提示词管理 + 用户管理（查看/修改其他用户信息） |
| `super_admin` | 超级管理员 | 管理员权限，且其信息仅可由本人修改 |

### 2. `roles` — 角色表

| `created_at` | `timestamptz` | `NOT NULL DEFAULT now()` | 创建时间 |
| `updated_at` | `timestamptz` | `NOT NULL DEFAULT now()` | 更新时间 |

- `roles_user_id_idx`：`user_id`

**`data` JSON 结构：**

| `spec` | `string` | `""` | 角色卡规范版本 |
| `name` | `string` | `""` | 角色名称 |
| `description` | `string` | `""` | 角色描述 |

### 5. `lorebooks` — 世界书表

| `created_at` | `timestamptz` | `NOT NULL DEFAULT now()` | 创建时间 |

### 8. `prompts` — 提示词表

存储可自定义的 LLM 提示词模板。

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `key` | `text` | `PRIMARY KEY` | 提示词唯一键（如 `story.system.base`） |
| `title` | `text` | `NOT NULL DEFAULT ''` | 标题 |
| `category` | `text` | `NOT NULL DEFAULT ''` | 分类（如 `story`） |
| `description` | `text` | `NOT NULL DEFAULT ''` | 描述 |
| `content` | `text` | `NOT NULL DEFAULT ''` | 提示词内容（支持 `{{变量}}` 模板） |
| `created_at` | `timestamptz` | `NOT NULL DEFAULT now()` | 创建时间 |
| `updated_at` | `timestamptz` | `NOT NULL DEFAULT now()` | 更新时间 |

**索引：**
- `prompts_category_idx`：`category`

**默认提示词：**

| key | title | 用途 |
|-----|-------|------|
| `story.system.base` | 故事生成系统提示 | 所有故事生成请求的 system 消息基础 |
| `story.scene.user` | 下一幕导演提示 | 生成下一幕 scene/narration/first |
| `story.opening.user` | 开场场景提示 | 生成第一轮的空场景 |
| `story.action.user` | 角色行动提示 | 生成单个角色的 action/dialogue |

## 数据迁移

系统启动时（`seed_legacy_json_data`）自动从 JSON 文件导入遗留数据：

| JSON 文件路径 | 目标表 | 说明 |
|--------------|--------|------|
| `data/users.json` | `users` | 用户数据（支持 `role` 字段） |
| `data/roles.json` | `roles` | 角色数据 |
| `data/preset.json` | `presets` | 预设数据 |
| `data/entries.json` | `entries` | 条目数据 |
| `data/lorebooks.json` | `lorebooks` | 世界书数据 |
| `data/stories/story_*.json` | `stories` + `story_rounds` | 故事数据 |

迁移规则：
- 仅当目标表无数据时执行迁移
- 用户已存在时跳过（按 username 匹配）
- 资源表使用第一个用户的 ID 作为 owner

## 关系图

```
users (1) ──→ roles (N)
users (1) ──→ presets (N)
users (1) ──→ entries (N)
users (1) ──→ lorebooks (N)
users (1) ──→ stories (N)
users (1) ──→ story_rounds (N)
stories (1) ──→ story_rounds (N)
```

所有资源表通过 `user_id` 外键关联到 `users` 表，级联删除。
